Models

Models are used to interact with the database. Here is an example of a model in TetaFramework:

class User extends Model {
    protected $table = 'users';
    protected $fillable = ['name', 'email', 'password'];
}

Views

Views are used to render HTML. Here is an example of a view in TetaFramework:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Welcome</title>
</head>
<body>
    <h1>Welcome, {{ name }}!</h1>
</body>
</html>

Controllers

Controllers handle the logic of your application. Here is an example of a controller in TetaFramework:

class UserController {
    public function show($id) {
        $user = User::find($id);
        return View::make('user.show', ['user' => $user]);
    }
}

Form Validation

Form validation is essential for ensuring data integrity. Here is an example of form validation in TetaFramework:

$validator = new Validator($request->all(), [
    'name' => 'required|max:255',
    'email' => 'required|email|unique:users',
    'password' => 'required|min:6',
]);

if ($validator->fails()) {
    return redirect('register')->withErrors($validator)->withInput();
}

Template

The template engine in TetaFramework allows you to create dynamic views. Here is an example:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Register</title>
</head>
<body>
    <div class="container">
        <h1>Register</h1>
        {{ @foreach ($errors as $error) }}
            <div class="error">{{ $error }}</div>
        {{ @endforeach }}
        <form method="POST" action="/register">
            <input type="text" name="name" value="{{ old('name') }}">
            <input type="email" name="email" value="{{ old('email') }}">
            <input type="password" name="password">
            <button type="submit">Register</button>
        </form>
    </div>
</body>
</html>

Language

The language system in TetaFramework allows for easy localization. Here is an example:

// In a language file (e.g., lang/en.php)
return [
    'welcome' => 'Welcome',
    'login' => 'Login',
];

Routes

Routes define the URL structure of your application. Here is an example of routing in TetaFramework:

Router::get('/', 'HomeController@index');
Router::post('/login', 'AuthController@login');

HTTP Classes

TetaFramework provides various HTTP classes to handle requests and responses. Here is an example:

// Handling a request
$request = new Request();
$name = $request->input('name');

// Sending a response
$response = new Response('Hello, world!');
$response->send();

Commands

Commands are used for running tasks from the command line. Here is an example:

class CreateUserCommand extends Command {
    protected $signature = 'user:create {name} {email}';

    public function handle() {
        $name = $this->argument('name');
        $email = $this->argument('email');
        User::create(['name' => $name, 'email' => $email]);
    }
}
